Human Task — Tier 1 Lab Content Compilation

ASEN 3501: Aerospace Experimental Methods

For: Test Engineers / Lab Development Team

Purpose

This document instructs test engineers to compile all content needed to support the Tier 1 (E1) lab assignments in ASEN 3501. The goal is to produce a complete, self-contained package for each Tier 1 experiment that can be handed to students — and that you yourself can run through as a student to verify it works.

All content for each lab comes from the lab catalog. This document tells you what to pull from it and what to do with it.

A parallel effort will be done for Tier 2 (E2) labs once Tier 1 is complete. The same checklist structure applies.

What Is Tier 1?

Tier 1 labs are the hands-on experiments used in E1 (Weeks 1–5). Students in E1 are learning to execute experiments, propagate uncertainty, and compare results to a predictive model — all for the first time in this course. The experiments in Tier 1 are designed to be accessible and concrete, so the supporting content needs to be especially clear and complete.

Step 2 — Content Checklist (Per Experiment)

For each Tier 1 experiment, locate and compile the following items from the lab catalog. Check off each item as it is confirmed present and of sufficient quality for student use. Note any gaps or issues in the "Status / Notes" column.

A. How to Conduct the Experiment

ItemStatus / Notes
Step-by-step procedure (setup through data collection)
Safety considerations specific to this experiment
Required equipment list (hardware, fixtures, connections)
Known setup pitfalls or common mistakes
Photographs or diagrams of the physical setup

B. Model — Solution / Derivation

ItemStatus / Notes
Governing equation(s) with symbols defined
Derivation or derivation reference (where does the model come from?)
Students will do: Assumptions and their justifications
Students will do: Expected output: what does the model predict, and in what units?
Students will do: Known limitations of the model (what does it not capture?)
Reference (textbook, paper, standard) if applicable

C. Sensor and DAQ Specifications

ItemStatus / Notes
List of all sensors used (name, manufacturer, model number)
For each sensor: accuracy / measurement uncertainty (from datasheet)
For each sensor: sampling rate capability and recommended rate for this experiment
For each sensor: measurement range (min/max)
DAQ system used (make, model, channel configuration)
DAQ resolution (bits) and resulting quantization uncertainty
Signal conditioning details (filtering, gain, anti-aliasing) if applicable
Datasheets (PDF or direct link) for each sensor and DAQ component

D. Sample Data

ItemStatus / Notes
At least one complete dataset from a prior run of this experiment
Data format documented (column headers, units, delimiter)
Conditions under which sample data was collected (temperature, setup notes, operator)
MATLAB script or template used to load and process the data (if available)
If applicable: known anomalies or quirks in the sample data

E. Anything Else in the Catalog

ItemStatus / Notes
Any additional instructor notes specific to this experiment
Prior student reports (for reference — do not use as answer key)
Known issues from prior runs that would affect a student's ability to complete the lab
Any content in the catalog not covered above that seems relevant

Step 3 — Run Through the Experiment as a Student

Once all checklist items are compiled, run through the entire Tier 1 experiment yourself using only the compiled content — no prior knowledge, no catalog access during the run.

The goal is to simulate the student experience. You should be able to:

  1. Read the procedure and set up the experiment without additional guidance
  2. Collect data using the documented sensor and DAQ configuration
  3. Analyze the data and propagate uncertainty using the documented specifications
  4. Compare your result to the model prediction using a named validation metric
  5. Write a brief summary of what worked, what was unclear, and what was missing

Document your experience in the Run-Through Notes section at the end of this document (Step 5).

Step 4 — Add Content If Needed

If the run-through in Step 3 reveals gaps — missing procedure steps, unclear sensor setup, no sample data for comparison — add the missing content now.

The compiled package must be complete enough for a junior-level undergraduate student with no prior lab experience to run the experiment successfully. If it is not, keep adding until it is.

Flag any additions you made in Step 4 — the instructor needs to review them before the content is considered final.
