Tier 1 Phase 2 — Prediction Memo (submitted before data release)

The pre-data-release commitment document. See `TIER1_assignment.md` for the full student-facing specification. Required sections: (1) predicted system response — specific values and ranges, expected trend shape committed in writing; (2) predicted uncertainty in the primary derived quantity — partial derivatives applied to the packet's sensor specs, result stated as estimate ± uncertainty; (3) predicted plot sketch — hand-drawn or MATLAB, with uncertainty bounds, labeled axes; (4) dominant uncertainty sources ranked by contribution; (5) acceptance criterion defined as a numerical threshold before data is seen; (6) risk assessment — at least one anomalous data pattern identified that would indicate a model-form limitation rather than measurement noise. Graded on quality of reasoning and completeness of commitment — not on whether the prediction turned out to be correct. Right prediction from wrong reasoning scores lower than wrong prediction from sound reasoning — this grading philosophy must be communicated explicitly in the rubric and in class before Phase 2 begins.

3 min Assessments 1 source variant(s)

Learning Objectives

  • Identify the five elements that make a prediction defensible — system response features, propagated uncertainty, a sketch with committed bounds, a ranked uncertainty source list, and a named acceptance criterion — and produce all five for an unfamiliar experiment — CLO 1, CLO 2, CLO 3
  • Distinguish between a vague prediction ("it will go up") and a committed prediction ("I expect the measured value to reach X ± Y by time T, because of physical reasoning Z") and articulate why the distinction matters for engineering credibility — CLO 3
  • Construct a predictive model with committed system response features and defined acceptance criterion for an experiment they did not run — CLO 1, CLO 3
  • Identify dominant uncertainty sources in an unfamiliar experimental setup and incorporate them into a prediction memo — CLO 2
