Part 1 — Spin Module Power-Up Procedure

Primary source: ASEN3801 Lab 3, Setup Procedure (pages 12–13); supplemental: ASEN3200 LabA1 (pages 8–9).

  1. Use the clamp provided to secure the spin module base to the lab station top. Place the clamp on the base plate at the spot labeled "clamp here".
  2. CHECK that you have completed step 1. If you do not clamp the spin module down and you run the motor, it will fly off the table and be damaged.
  3. Connect the USB cable Type B end to the National Instruments myRIO port labeled "DEVICE" (the middle port). The other Type A end connects to the lab station computer. (Do not connect to the Type A top port on the myRIO.)
  4. Connect the power adapter to the barrel connector. The power light on the myRIO should now be illuminated blue.
  5. Wait 1–2 minutes until the Status light on the myRIO turns from orange to off.
  6. Verify the lab assistant has connected the second power supply to the base station printed circuit board (PCB). Verify the U8002A power supply is set to a limit of 12.00 V and 0.300 A (press "Display Limit" on the supply). Do not plug in any power connections until power supply settings have been verified.
  7. Confirm that the satellite 9V battery pack has been plugged in. Wait for the lower board motor controller to flash its green LED.
  8. Once the above steps are complete, turn on the satellite board power located on the top PCB, inside the spin module chassis, using the red toggle switch.
  9. Verify the board goes through a series of status LEDs shown near the toggle switch. The remaining LED under the switch should be Yellow; the Alive LED should be green.
  10. Open All_Spin_Module_Labs.vi located in your current semester's class folder (Courses H:/AES/Software/VIs/ASEN 3200/). Do NOT run the VI yet.
  11. Check the battery voltage meter in the VI status panel (bottom right corner). If the battery voltage of the satellite's onboard battery pack drops below 14 V, turn off the top satellite board with the red toggle switch and request staff assistance for new batteries.
  12. The "Reinitialize Arduino" button is used if you notice the gyro voltage drift over time from zero at rest. Use it to reset this offset bias.
  13. The green indicators show if the myRIO (on the base) and the Arduino DUE (inside the satellite) are properly connected and communicating.
  14. Check the "Link Speed" dial — a good typical value is 1.5 or higher. If you see frequent dropouts to zero, the Xbee channel may be experiencing interference from other wireless sources, or the battery voltage may be too low.

NOTE: For safety, you cannot change the LabVIEW operating modes (Torque vs. Gyro) of the Spin Module while the VI is running.

Part 2 — Rate Gyro Calibration

Primary source: ASEN3801 Section 3.1; supplemental: ASEN3200 LabA1 Section 5.1.

Purpose: Characterize the MEMS gyro bias (b) and scale factor (K) relative to the encoder truth reference.

Sub-Part 2a: Manual Experiment (Qualitative)

  1. Select the "Gyro" tab on the VI front panel.
  2. Start running the VI by clicking "play". The VI will now display live gyro measurements.
  3. Select "Manual Control" from the VI front panel.
  4. Move the spacecraft gently back and forth by hand in a smooth, sinusoidal manner so that a good linear relationship between gyro rate measurement and input angular rate can be observed on the VI.
  • Note: Abrupt changes in angular velocity lead to non-linear effects in the gyro output that appear as "looping" in the relationship between true angular rate and gyro measurement. Use gentle, sinusoidal motion to avoid this.
  1. Observe how moving the spacecraft at different rates affects the data collection.
  2. Collect approximately 1 minute of data for observation; there is no need to save this file.
  3. Take notes on the performance of the MEMS gyro for later discussion.

Sub-Part 2b: Motor-Driven Calibration Data Collection

  1. While still on the "Gyro" tab, select "Automatic RW Control" from the VI front panel.
  2. Set the desired output frequency and amplitude. A 0.2 Hz sine wave at an amplitude of 0.5 Amps is a good starting point.
  3. Click "Begin Test". The VI will display data on the screen (not yet logging to file).
  4. Collect approximately 1 minute of data.
  5. Click "Write to File" and save using the naming convention:

YEAR_MONTH_DAY_SEC_MEMS_F#C#

  • Example: 2024_XX_XX_001_MEMS_F02C05 (Section 001, 0.2 Hz, 0.5 A)
  1. Repeat for at least 2 more different frequency/amplitude combinations, saving each file.
  2. Use the "STOP" button when done collecting.

Data quality note: If "DUMPING" is displayed and illuminated (rather than "QUEUING"), data will not be saved — check this before clicking Write to File.

Sub-Part 2c: MATLAB Analysis

From ASEN3801 Section 3.1, Tasks 1–4.

  1. For one data set: import the data into MATLAB. Identify what information is in each column and convert the rate data from RPM to rad/s.
  2. On the same graph, plot the time history of the encoder rate measurement (true angular rate) and the gyro rate measurement (in rad/s). Note: you should observe that the gyro rate measurements have the opposite sign from the encoder rate measurements.
  3. To determine the calibration parameters, plot the encoder rate measurement on the horizontal axis and the gyro rate measurement on the vertical axis. Fit a line to the data to determine:
  • Gyro bias b [rad/s] (y-intercept)
  • Scale factor K [unitless, (rad/s)/(rad/s)] (slope)
  1. Write a calibration equation for how the gyro rate measurements should be corrected. Apply the calibration to the data set and plot the time history of encoder rate and calibrated rate on the same axes — they should match well.
  2. Make a table of b and K for each run. Compute the mean and standard deviation of b and K across all runs. Explain which trials you used and which portions of the data, if any, you removed before computing statistics.
  3. For two data sets, compute and plot:
  • (a) Time history of encoder rate and calibrated rate
  • (b) Time history of angular rate error (calibrated rate − encoder rate)
  • (c) Time history of true angular position (from encoder) and measured angular position (from calibrated gyro rate) — compute positions by integrating respective rates
  • (d) Time history of angular position error (measured − true)
  • (e) Angular position error as a function of encoder rate (encoder rate on x-axis)

Model / Derivation

Theoretical Derivation — Spin Modules: Rate Gyro Calibration and Inertia Estimation

> Scope: MEMS rate gyro sensor model and calibration; reaction wheel MOI estimation; spacecraft body MOI estimation > Sources: ASEN3801 Lab 3 Sections 3.1–3.3a; ASEN3200 LabA1 Sections 5.1–5.2, Section 6

Section 1 — MEMS Rate Gyro: Sensor Model and Calibration

Physical Sensor Description

A MEMS (Micro-Electro-Mechanical System) gyroscope measures angular rate about one sensitive axis. The rate gyro installed on the spin module is mounted with its sensitive axis pointed upward, measuring spacecraft rotation about the vertical spin axis. The gyro output is proportional to the true angular rate but typically has two systematic errors:

  • Bias (b): a constant offset — the sensor reads a nonzero rate even when the spacecraft is stationary.
  • Scale factor error (K): the ratio between the gyro's output and the true angular rate deviates from 1.0 — the sensor over- or under-reports the rate magnitude.

Calibration Model

The linear calibration model for the MEMS gyro is:


gyro_output = K · ω_true + b + noise

Where:

  • gyro_output — measured gyro rate [rad/s]
  • ω_true — true angular rate of spacecraft [rad/s], provided by base encoder
  • K — scale factor [unitless; (rad/s)/(rad/s)]
  • b — bias [rad/s]
  • noise — zero-mean random sensor noise (not modeled)

Note on sign convention: In practice, the gyro output has the opposite sign relative to the encoder rate measurement. The linear regression procedure (fitting gyro output vs. encoder rate) handles this automatically — K will be negative when this sign flip is present.

Calibration Procedure (Parameter Identification)

The encoder (on the base motor) provides a truth reference for spacecraft angular velocity. To determine K and b:

  1. Collect data at multiple spacecraft angular velocities driven by the reaction wheel motor (sinusoidal excitation).
  2. Plot encoder rate (x-axis) vs. gyro rate measurement (y-axis).
  3. Fit a line to the data using linear regression:
  • Slope = K
  • Intercept = b
  1. Write the calibration equation:

ω_calibrated = (gyro_output − b) / K

The calibrated rate should closely match the encoder rate.

Calibration Performance Metrics

Students should characterize the following after completing multiple calibration runs:

MetricDefinitionSignificance
Bias repeatabilitystd(b) across runsIs bias consistent enough to use a single calibration?
Scale factor precisionstd(K) across runsHow well-determined is K?
Useful rate rangeRate below which the gyro deviates from linearityUpper limit for reliable gyro use
Nonlinear behavior"Looping" in gyro vs. encoder plotIndicates saturation-like behavior at abrupt changes

Angular Position from Gyro Integration

Given calibrated angular rate, angular position can be estimated by integration:


θ(t) = θ(0) + ∫ ω_calibrated(t) dt

In MATLAB, this is computed as cumulative trapezoidal integration:


theta = cumtrapz(t, omega_calibrated);

Key finding — position drift: Because any residual bias error after calibration accumulates over time during integration, the estimated angular position will drift from the true position. This drift:

  • Grows monotonically with time
  • Is proportional to the residual rate error
  • Is a fundamental limitation of gyro-only navigation (no drift correction without additional sensor)

Students should observe and explain this mechanism by comparing the integrated gyro position estimate to the true position derived from the encoder.

Section 2 — Reaction Wheel and Spacecraft Body: MOI Estimation

Physical Basis

The moment of inertia (MOI) of a rotating body can be measured experimentally by applying a known, constant torque and measuring the resulting angular acceleration. This follows directly from Newton's second law for rotation:


τ = I · α

Rearranging to solve for MOI:


I = τ / α [kg·m²]

Where:

  • τ = applied torque [N·m]
  • I = moment of inertia about the spin axis [kg·m²]
  • α = angular acceleration [rad/s²]

This same method applies to two different objects in this lab:

  • Reaction wheel: Hold the spacecraft body stationary; apply torque to RW motor; measure RW angular velocity via Hall sensors → estimate I_RW
  • Spacecraft body: Allow spacecraft to spin freely; apply torque to base motor; measure spacecraft angular velocity via base encoder → estimate I_sc

Using the same method on two different objects makes the method itself the focus, rather than a specific piece of hardware.

Angular Acceleration Estimation

Under constant applied torque, angular velocity increases linearly with time:


ω(t) = ω₀ + α · t

Therefore, angular acceleration α is the slope of a linear fit to ω(t) data.

In MATLAB:


p = polyfit(t, omega, 1); % 1st-order polynomial fit
alpha = p(1); % slope = angular acceleration [rad/s^2]

Current-to-Torque Conversion

The LabVIEW VI measures motor current I_motor [A]. Torque is computed from the motor's torque constant:


τ = k_t · I_motor

Where:

  • k_t = motor torque constant [N·m/A or mN·m/A] from the motor datasheet
  • I_motor = measured motor current [A] from VI

Two different k_t values are required:

  • Reaction wheel motor: part #251601 (datasheet page 300 of motor catalog)
  • Base motor: part #200142 (datasheet page 265 of motor catalog)

[MISSING: Actual k_t values for both motors — motor datasheets are embedded as images in the ASEN3801 source document (pages 14–15). The torque calculation and MOI estimation cannot be completed without these values. See instrumentation/datasheets/README_agent.md.]

Statistical Characterization of MOI Estimate

Because individual trials may have minor variations in torque delivery, students should perform multiple trials and report a statistical estimate:

  • Collect at least 5 trials at different torque values
  • For each trial: compute I = τ / α
  • Report:
  • I_mean = mean(I₁, I₂, ..., I_N)
  • I_std = std(I₁, ..., I_N)
  • I_unc = I_std / sqrt(N) (uncertainty of the mean)

Students should also explain which trials they used and why (data quality judgment: did any trial show clearly non-constant torque or poor linear fit quality?).

Angular Momentum Capacity (Reaction Wheel)

The maximum angular momentum the reaction wheel can store:


H_max = I_RW · ω_max [N·m·s]

Where ω_max = 4000 rpm = 4000 × (2π/60) = 418.9 rad/s (from ASEN3801 source).

Reaction Wheel Saturation Time

The time before the reaction wheel reaches its angular velocity limit under a constant external disturbance torque:


t_sat = H_max / τ_disturbance [s]

Using a representative low Earth orbit aerodynamic drag torque:

  • τ_disturbance = 10⁻⁴ N·m (given in source as representative LEO disturbance)

Sensor and Instrumentation Summary — Spin Modules: Rate Gyro Calibration and Inertia Estimation

> Sources: ASEN3801 Lab 3; ASEN3200 LabA1 > Scope: In-scope instrumentation only (rate gyro calibration; RW MOI; spacecraft body MOI)

Hardware System Overview: CU Boulder Spin Module

The Spin Module is a single-axis spacecraft mockup developed at CU Boulder. It consists of two mechanically distinct sections connected only by a motor shaft:

Upper section ("satellite"):

  • Self-contained; runs on onboard battery
  • Contains: MEMS rate gyro (IMU), reaction wheel with brushless DC motor, 9V battery pack, Arduino DUE microcontroller, Xbee wireless radio
  • Communicates wirelessly with the base via Xbee

Lower section ("base"):

  • Connected to lab power supply
  • Contains: brushless DC base motor with optical encoder, NI myRIO DAQ hardware, power PCB
  • Communicates via USB to lab computer

The only mechanical connection between upper and lower sections is the motor shaft, allowing the spacecraft to rotate freely or be spun by the base motor.

4. Adaptation Notes for ASEN 3501

What Maps Cleanly to 3501

The in-scope content (rate gyro calibration + MOI estimation) maps directly to ASEN 3501's E1 phase and connects to ASEN V&V 10-2006 sensor characterization concepts:

Rate gyro calibration as sensor characterization: This is a complete, self-contained sensor characterization exercise:

  • Determine K and b
  • Quantify repeatability across runs (std dev of K and b)
  • Understand fundamental limitation (position drift from integrated bias)
  • Maps strongly to CLO 4 (instrumentation) and CLO 2 (statistics)

MOI estimation as model-based parameter identification: The MOI experiment is a clean example of using Newton's 2nd Law as the underlying model and experimental data to estimate a physical parameter (I). This is a direct application of the AMVF "experimentally informed model" concept. Using the same method on two different objects (reaction wheel and spacecraft body) keeps the method itself in focus — good ASEN 3501 pedagogy.

Position drift analysis: The observation that integrated gyro errors accumulate over time (position drift grows with time and rate) is conceptually rich and connects to navigation, attitude estimation, and sensor fusion. This is a strong discussion prompt for E1.

Considerations Before Running in 3501

  1. Motor torque constants (CRITICAL): Without k_t for both motors, the torque calculation and MOI computation cannot be completed. This is the single most important item to resolve. See instrumentation/datasheets/README_agent.md for action steps.
  1. Binary documentation files: SpinModule.pdf, SpinModule1.docx, and SpinModule2.docx in the source directory have not been read. They likely contain hardware diagrams, wiring details, or sensor specifications that would substantially improve the instrumentation section.
  1. Sample data files: No archived data files from prior lab runs exist in the source directory. Sample files would be valuable for 3501 analysis template development and instructor reference.
  1. Block diagram task: Both source documents ask students to draw a functional block diagram of the experimental setup. This is a useful addition to 3501 if retained — it reinforces CLO 4 and forces students to map sensor/actuator/DAQ connections before collecting data.
  1. Out-of-scope content is well-documented and clean: The control content (PD control design, ASEN3801 Section 3.3b-d) spans a large portion of the ASEN3801 source but is clearly separable from the in-scope content. The boundary is sharp — no bleed-through of control concepts into the calibration and MOI sections.
